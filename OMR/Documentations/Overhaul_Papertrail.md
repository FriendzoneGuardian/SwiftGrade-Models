## What Was Implemented

1. Dynamic thresholding foundation
- Added largest-gap threshold engine (`dynamic_thresholds.py`) with confidence/fallback logic.

2. Two-pass classifier
- `run_classifier.py` now:
  - extracts per-bubble diagnostics in pass 1,
  - applies static or dynamic decision policy in pass 2,
  - supports strict FILLED confirmation gate.

3. Decision diagnostics
- `butler_preprocessor.py` now emits:
  - `variance`,
  - `solidity_overall`,
  - `center_fill_ratio`,
  - `border_fill_ratio`,
  - `border_to_dark_ratio`,
  - subtraction/variance gate flags.

4. Optional Levels normalization
- LUT-based normalization added behind `preprocessor.levels.enabled`.

5. Run papertrail artifacts
- Generated by classifier when enabled:
  - `papertrail/audit_metrics.csv`
  - `papertrail/threshold_report.json`
  - `papertrail/run_summary.json`

6. Evaluation harness
- `evaluate_classifier.py` computes confusion, FILLED precision/recall/F1/FPR from audit logs with optional ground truth csv.

7. Notebook hardening for later ML execution
- Added path fallback config, class-balance weights, precision-first training metrics, and notebook papertrail outputs under `Phase2/papertrail/notebook/`.

## Most Likely Breakpoints

1. Missing dataset paths
- Classifier defaults to `Phase2/dataset_sample_10pct`.
- Notebook defaults to `ModelBackEnd/SwiftGrade_Datasets` then fallback candidates.

2. Dynamic threshold confidence low
- If score distribution is flat, dynamic mode falls back to configured threshold.
- Check `threshold_report.json` for `reason` and `confident`.

3. Over-strict FILLED gate
- If FILLED counts collapse, inspect `strict_filled_reason` in `audit_metrics.csv`.

4. Local-group dynamic override not used
- If no group CSV is provided or mapping fails, only global dynamic threshold is used.

5. Master blank not found
- Subtraction gracefully disables; see preprocessor warning and `used_template_subtraction` column.

6. Notebook not runnable in this environment
- This unit is ML-incapable by design; notebook edits are for later execution.

## Not Yet Implemented (Planned in Overhaul)

1. Geometry stage wiring in runtime
- Marker-first correction, contour fallback, ORB rescue are assessed and planned but not yet integrated in Phase2 crop classifier runtime.

2. Row-level arbitration module
- Multi-mark and erasure conflict resolution script is still pending implementation.

3. Full processor-manager migration
- Plugin orchestration pattern is documented and planned; current runtime remains script-driven with backward-compatible entry points.

## Debug Checklist

1. Confirm config mode:
- `thresholds.method` is either `static` or `dynamic_largest_gap`.

2. Inspect audit rows for false positives:
- filter `predicted_label == FILLED` and compare against truth.
- inspect `decision_reason`, `border_to_dark_ratio`, and `solidity_overall`.

3. Compare static vs dynamic quickly:
- Run once with `thresholds.method=static`, once with `dynamic_largest_gap`.
- Compare FILLED precision and FPR in evaluation report.

4. If many `strict_reject:*` rows:
- relax `decision.strict_filled` thresholds incrementally.

## Change Safety

- Existing script entry point remains: `run_classifier.py`.
- Config defaults preserve prior behavior unless dynamic mode is explicitly used.
- Notebook changes are non-executed and forward-compatible.
